VAE in S4TF !!

by Vaibhav Balloli

2020-03-22

It's been 3 months since my first open-source contribution to the S4TF community in the Swift APIs repository and I'm really excited to keep working on this!! S4TF's fantastic high-level API for differentiable programming is what has got me hooked onto this and through this post, I hope on giving back to the community.

Variational Autoencoders (Kingma et al.[1]) have been around for a lot of time and there are many awesome explanations online regarding this. I'll be linking to a few of them, but through this post, I'd like to highlight important Swifty features that S4TF supports and how you can start contributing to this incredible repository!!

The entire code can be found here: https://github.com/vballoli/swift-models/blob/VAE/Autoencoder/VAE1D/main.swift

Step 1: Define the autoencoder

  1. Declare and initialize the model layers
  2. callAsFunction() lets you 'call ' your struct object - where both input and output must be of type Diiferenitable. Here, we input an image tensor and return an Array of Tensors. Pay attention to the return type: [Tensor<Float>]. When import TensorFlow is executed, it extends Swift's Array implementation to make it differentiable. That is so cool !!  

Step 2: Loss function

This is relatively simple. The loss function's value must be scalarized, and since all the intermediate operations and functions are differentiable, the entire function becomes differentiable.

Step 3: Train !!

Load dataset, set hyperparameters, that's a go on training !!

And that's it!! Training a 1 dimensional AutoEncoder is that simple in Swift for TensorFlow. More such tutorials can be found on their official website - S4TF. You must now be wondering, "That's awesome! How do I start contributing !?". Well, it's pretty intuitive:

1. Write your code(duh)

2. Tell Swift to compile your code by adding you main directory's path to Package.swift - based on what you've implemented, either a simple run by adding in the targets section - e.g. .target(name: "VariationalAutoencoder1D", dependencies: ["Datasets", "ModelSupport"], path: "Autoencoder/VAE1D") in the targets array and run it as swift run -c release VAE1D or  if it is a library like code, add it to products - e.g. .library(name: "Name", targets: ["Name"]) and their corresponding targets .target(name: "Name", path: "Name/Name2/Name3").

3. Run swift build and you're all set.

4. Send a pull request and onto next ones !!

Looking forward to comments on this and stay tuned for more.

References
